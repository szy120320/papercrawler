04 · Two-Stage Filtering

两阶段打分架构

两个独立分数各自走自己的门限,不合并成一个 final score。避免"方法对领域错"的论文误判。

Stage 1 · 粗筛
DomainFilter · 基于 title
coarse_score = 0.6 · [must_have 命中]
+ 0.1 · [should_have 命中数] (cap 0.3)
− 0.5 · [exclude 命中]
+ 0.05 · [category 命中数] (cap 0.20)
  • 只用 title,不需要 abstract,速度极快
  • 粗筛过滤后,后续 enrich 和精筛只对通过论文跑
  • 门限默认 0.7(可在 CLI / TOML 覆盖)
Stage 2 · 细筛
SemanticFilter · 命中计数
semantic_score = count(kw ∈ semantic_keywords
∧ match(kw, title + abstract + keywords))
保留条件: semantic_score ≥ semantic_min_matches(默认 3)
  • 同时看 title + abstract + keywords 三段文本
  • 模糊匹配:大小写不敏感 + difflib 字符相似度
  • 命中数 ≥ 3 才通过(用户可配置)

双门限:粗筛 ≥ 0.7 且 命中数 ≥ 3。两个独立分数各自判断,不计算合并分数。 CSV 中 coarse_score 列是浮点,semantic_score 列是整数(命中数)。

06 · Sci-Hub Anti-Crawl Bypass

Sci-Hub 兜底链

所有合法渠道都失败后,Sci-Hub 是最后兜底。两层反爬必须绕过:

DDoS-Guard
第一层反爬
真浏览器通过
ALTCHA PoW
第二层反爬
浏览器 Web Crypto 算 nonce
论文页
跳到 /<doi>
提取 <object data>
PDF
用 page.evaluate
(fetch) 下载字节

关键技术点:scihub_downloader.py 用 Playwright + 系统 Edge / Chrome, 不依赖老旧的 scidownl 库(那个库被 DDoS-Guard 全拦截)。 浏览器内的 fetch() 拿 PDF 字节(不是 httpx)— httpx 会被 DDoS-Guard 二次拦截。
